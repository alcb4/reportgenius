── Backend route ────────────────────────────────────────

  DELETE /api/v1/account

  Cascade delete in this order (respect FK constraints):
    1. report_text / generated_reports
    2. ratings
    3. sessions
    4. students
    5. classes
    6. organisation members
    7. organisation
    8. user account

  Use prisma.$transaction for atomicity — all or nothing.

  email ("Your account and all data has been deleted.")

  After deletion: invalidate all sessions/JWTs for that user,
  return 200, frontend redirects to /login with message.

ITEM 4 — DATA RETENTION POLICY (automated)
═══════════════════════════════════════════════════════

Retention schedule agreed:
  Sessions + reports:  academic year + 1 year
  Student ratings:     same
  Account data:        while active
  Post-deletion:       hard delete, 30 day grace (optional)

── Schema changes ───────────────────────────────────────

  Add to relevant tables if not present:
    created_at   DateTime  @default(now())
    updated_at   DateTime  @updatedAt

  Add to organisation or user table:
    deleted_at   DateTime?   // null = active, set = pending deletion

── Cron job (Vercel Cron) ───────────────────────────────

  vercel.json:
    {
      "crons": [{
        "path": "/api/cron/cleanup",
        "schedule": "0 2 * * *"    // 2am daily
      }]
    }

  Route: src/app/api/cron/cleanup/route.ts

    export async function GET(req: NextRequest) {
      // Verify cron secret to prevent public triggering:
      const authHeader = req.headers.get('authorization')
      if (authHeader !== `Bearer ${process.env.CRON_SECRET}`) {
        return NextResponse.json({ error: 'Unauthorized' }, { status: 401 })
      }

      const now = new Date()

      // 1. Hard delete accounts marked deleted > 30 days ago:
      const cutoff30 = new Date(now)
      cutoff30.setDate(cutoff30.getDate() - 30)

      await prisma.organisation.deleteMany({
        where: {
          deleted_at: { lte: cutoff30 }
        }
      })
      // Cascade handles users, classes, students, sessions etc.

      // 2. Delete sessions older than current year + 1:
      const cutoffSessions = new Date(now)
      cutoffSessions.setFullYear(cutoffSessions.getFullYear() - 1)
      cutoffSessions.setMonth(6) // July (end of academic year)

      await prisma.reportSession.deleteMany({
        where: {
          created_at: { lte: cutoffSessions }
        }
      })

      return NextResponse.json({ ok: true, ran: now.toISOString() })
    }

  Add to .env:
    CRON_SECRET="<random 32 char string>"

  Add to vercel env vars too.
